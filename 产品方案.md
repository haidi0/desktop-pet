# 桌面宠物产品方案

## 一、产品定位

**名称**：泽小安桌面宠物
**平台**：Windows 10/11
**技术栈**：C# WPF (.NET 8)
**目标用户**：喜欢可爱桌面装饰、需要轻量互动陪伴的用户

## 二、核心功能

### 2.1 基础显示
- 透明窗口，只显示角色形象
- 置顶显示，不被其他窗口遮挡
- 启动时显示在屏幕右下角

### 2.2 交互功能
| 交互方式 | 功能 |
|---------|------|
| 左键拖拽 | 移动宠物位置，松开后边缘吸附 |
| 左键单击 | 触发互动动画（挥手、比心等） |
| 右键单击 | 打开功能菜单 |

### 2.3 动画系统
| 状态 | 动画 | 触发条件 |
|------|------|---------|
| 待机 | 呼吸、眨眼 | 默认状态 |
| 行走 | 左右移动 | 随机触发（30-120秒） |
| 睡眠 | 闭眼、轻微晃动 | 5分钟无交互 |
| 拖拽 | 被拎起的表情 | 拖拽过程中 |
| 互动 | 挥手、比心、跳舞 | 点击触发 |

### 2.4 系统托盘
- 最小化到托盘
- 托盘图标右键菜单：
  - 显示宠物
  - 切换角色
  - 设置
  - 退出

### 2.5 设置功能
- 宠物缩放比例
- 开机自启动
- 窗口置顶开关
- 角色切换

### 2.6 多角色支持
- 角色文件夹化管理
- 动态扫描可用角色
- 切换角色无需重启

## 三、角色资源规范

```
assets/characters/
└── zexiaoan/
    ├── character.json      # 角色配置
    ├── icon.png            # 托盘图标
    ├── idle/               # 待机动画
    │   ├── frame_001.png
    │   └── frame_002.png
    ├── walk_right/         # 向右行走
    ├── walk_left/          # 向左行走
    ├── sleep/              # 睡眠动画
    ├── drag/               # 拖拽动画
    └── interact/           # 互动动画
```

**图片规格建议**：
- 尺寸：200x300 像素（可根据角色调整）
- 格式：PNG（透明背景）
- 帧数：每状态 5-20 帧

## 五、用户体验流程

```
启动 → 显示在右下角 → 播放待机动画
  │
  ├─→ 用户拖拽 → 跟随移动 → 松开吸附边缘
  │
  ├─→ 用户点击 → 播放互动动画 → 回到待机
  │
  ├─→ 30-120秒无操作 → 随机行走 → 回到待机
  │
  ├─→ 5分钟无操作 → 进入睡眠 → 点击唤醒
  │
  └─→ 右键菜单 → 切换角色/设置/退出
```

## 六、竞品分析

| 产品 | 优点 | 缺点 |
|------|------|------|
| 桌面宠物猫 | 动画丰富、交互多 | 功能臃肿、资源占用高 |
| Shimeji-ee | 开源、可扩展 | 需要Java、配置复杂 |
| 人工桌面 | 美观、官方支持 | 仅限官方角色 |
| 猫猫桌面宠物 | 轻量、可爱 | 功能单一 |

**我们的差异化**：
1. 角色可定制（用户自制角色包）
2. 轻量级（纯WPF，无依赖）
3. 专注陪伴感（心情系统、状态变化）
